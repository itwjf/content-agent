## 2. 模块划分

### 2.1 互动理解模块 (interaction_module)
- **输入**：弹幕消息列表、情绪分析数据
- **处理**：语义分析、意图识别、高频问题聚合、优先级标记
- **输出**：结构化用户意图列表，含优先级评分

### 2.2 卖点拆解模块 (selling_point_module)
- **输入**：商品数据、用户意图列表
- **处理**：参数→利益点转换、卖点与问题匹配、LLM 生成话术
- **输出**：针对性卖点话术

### 2.3 合规约束模块 (compliance_module)
- **输入**：待输出文本
- **处理**：违禁词扫描、替换建议生成
- **输出**：合规检查结果 + 替代方案

### 2.4 内容结构引擎 (structure_engine)
- **输入**：直播状态（阶段、时长）
- **处理**：阶段判断、时间规划、下一步提示
- **输出**：当前阶段指令

### 2.5 决策融合模块 (decision_module)
- **输入**：各模块输出
- **处理**：优先级打分、冲突处理、最终排序
- **输出**：最终提词指令 JSON

## 3. 数据流转

```
多模态输入
    │
    ▼
互动理解模块 ──→ 提取意图、高频问题
    │
    ├──→ 卖点拆解模块 ──→ 匹配话术
    │
    ├──→ 内容结构引擎 ──→ 阶段指令
    │
    └──→ 合规约束模块 ──→ 过滤检查
              │
              ▼
         决策融合中枢
              │
              ▼
         提词生成模块
              │
              ▼
         主播提词器输出
```

## 4. 关键接口说明

### POST /api/v1/agent/decide
Agent 核心决策接口

**请求体**：见 `api.md`
**响应**：提词指令 JSON

### POST /api/v1/products
商品信息管理

### GET /api/v1/compliance/check
合规检查

## 5. 优先级打分机制

| 触发条件 | 基础分 |
|---------|--------|
| 弹幕高频问题（≥3次） | 90 |
| 负面情绪反馈 | 80 |
| 购买意向信号 | 85 |
| 阶段切换提示 | 70 |
| 普通卖点推送 | 50 |

冲突处理规则：取最高分模块输出，其余降级为备选。
