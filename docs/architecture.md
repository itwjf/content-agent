## 2. 模块划分

### 2.1 互动理解模块 (interaction_module)
- **输入**：弹幕消息列表、情绪分析数据
- **处理**：语义分析、意图识别、高频问题聚合、优先级标记
- **输出**：结构化用户意图列表，含优先级评分

### 2.2 卖点拆解模块 (selling_point_module)
- **输入**：商品数据、用户意图列表
- **处理**：参数→利益点转换、卖点与问题匹配、LLM 生成话术
- **输出**：针对性卖点话术

### 2.3 合规约束模块 (compliance_module)
- **输入**：待输出文本
- **处理**：违禁词扫描、替换建议生成
- **输出**：合规检查结果 + 替代方案

### 2.4 内容结构引擎 (structure_engine)
- **输入**：直播状态（阶段、时长）
- **处理**：阶段判断、时间规划、下一步提示
- **输出**：当前阶段指令

### 2.5 决策融合模块 (decision_module)
- **输入**：各模块输出
- **处理**：优先级打分、冲突处理、最终排序
- **输出**：最终提词指令 JSON

## 3. 数据流转

```
多模态输入
    │
    ▼
互动理解模块 ──→ 提取意图、高频问题
    │
    ├──→ 卖点拆解模块 ──→ 匹配话术
    │
    ├──→ 内容结构引擎 ──→ 阶段指令
    │
    └──→ 合规约束模块 ──→ 过滤检查
              │
              ▼
         决策融合中枢
              │
              ▼
         提词生成模块
              │
              ▼
         主播提词器输出
```

## 5. 优先级打分机制

| 触发条件 | 基础分 |
|---------|--------|
| 弹幕高频问题（≥3次） | 90 |
| 负面情绪反馈 | 80 |
| 购买意向信号 | 85 |
| 阶段切换提示 | 70 |
| 普通卖点推送 | 50 |

冲突处理规则：取最高分模块输出，其余降级为备选。

## 7. 模块间交互详情

### 7.1 数据流向
1. **前端 → 后端**：直播状态、弹幕数据、商品信息、后台数据
2. **后端内部**：
   - 互动理解模块 → 卖点拆解模块：用户意图和高频问题
   - 互动理解模块 → 内容结构引擎：直播状态和时长
   - 卖点拆解模块 → 合规约束模块：生成的话术
   - 各模块 → 决策融合模块：处理结果和优先级
   - 决策融合模块 → 提词生成模块：最终决策
3. **后端 → 前端**：提词指令和直播结构信息

### 7.2 调用关系
- **Agent决策接口**：协调所有模块的调用流程
- **RAG服务**：为卖点拆解模块提供语义检索支持
- **LLM服务**：为卖点拆解模块提供话术生成支持
- **合规服务**：为提词生成模块提供合规检查

## 8. 扩展性与性能考虑

### 8.1 扩展性
- **模块化设计**：各模块独立封装，可单独升级
- **插件架构**：支持新增模块和替换现有模块
- **配置驱动**：通过配置文件调整系统行为

### 8.2 性能优化
- **异步处理**：使用FastAPI的异步特性处理并发请求
- **缓存机制**：缓存热点数据和计算结果
- **批处理**：对弹幕数据进行批处理减少API调用
- **向量索引**：优化Qdrant向量搜索性能

### 8.3 容错机制
- **服务降级**：当LLM或Qdrant不可用时，启用降级策略
- **重试机制**：对外部API调用实现自动重试
- **监控告警**：实时监控系统状态，及时发现问题
