# ContentAgent - 智能直播辅助 AI Agent 系统

## 项目简介

ContentAgent 是一款基于 AI 大模型及 Agent 架构的智能直播辅助系统，专为直播电商场景设计。系统能够实时接收多模态输入（弹幕、直播阶段、后台数据），通过多模块协同推理，生成精准、实时、可执行的提词建议，辅助主播进行结构化、高转化且合规的直播。

## 功能特点

- **实时互动理解**：实时分析弹幕内容，识别用户意图、高频问题和情绪变化
- **智能卖点拆解**：将商品参数转化为利益点，匹配用户问题生成针对性话术
- **内容结构引擎**：自动划分直播阶段，实时提示当前阶段及下一阶段准备
- **合规约束保障**：内置违禁词库（100+），实时过滤并提供合规替代方案
- **智能决策融合**：基于多源输入进行优先级打分，生成最优提词指令
- **RAG 知识库**：整合产品信息文档，提供精准的产品知识支持
- **Docker 快速部署**：提供完整的容器化部署方案，一键启动所有服务

## 核心模块

### 1. 互动理解模块
- **输入**：弹幕消息列表、情绪分析数据
- **处理**：语义分析、意图识别、高频问题聚合、优先级标记
- **输出**：结构化用户意图列表，含优先级评分

### 2. 卖点拆解模块
- **输入**：商品数据、用户意图列表
- **处理**：参数→利益点转换、卖点与问题匹配、LLM 生成话术
- **输出**：针对性卖点话术

### 3. 合规约束模块
- **输入**：待输出文本
- **处理**：违禁词扫描、替换建议生成
- **输出**：合规检查结果 + 替代方案

### 4. 内容结构引擎
- **输入**：直播状态（阶段、时长）
- **处理**：阶段判断、时间规划、下一步提示
- **输出**：当前阶段指令和下一阶段准备建议

### 5. 决策融合模块
- **输入**：各模块输出
- **处理**：优先级打分、冲突处理、最终排序
- **输出**：最终提词指令 JSON

## 快速开始指南

### 1. 克隆项目

```bash
git clone git@github.com:itwjf/content-agent.git
cd content-agent
```

### 2. 配置环境变量

```bash
# 复制配置模板
cp backend/.env.example backend/.env

# 编辑 .env 文件，填入以下配置：
# - LLM_API_KEY: 大模型 API 密钥
# - SILICONFLOW_API_KEY: 向量生成 API 密钥（可选）
# - 其他配置项（根据需要调整）
```

### 3. 启动服务

#### 方式一：Docker Compose（推荐）

```bash
# 进入 Docker 配置目录
cd docker

# 启动所有服务（首次启动会构建镜像，需要几分钟）
docker-compose up -d

# 查看服务状态
docker-compose ps
```

#### 方式二：本地开发

```bash
# 启动后端服务
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --reload

# 启动前端服务（新终端）
cd frontend
npm install
npm run dev
```

### 4. 访问服务

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端 | http://localhost:3000 | 直播模拟控制台 |
| 后端 API | http://localhost:8000 | FastAPI 服务 |
| API 文档 | http://localhost:8000/docs | Swagger 交互式文档 |

## 核心 API

### POST /api/v1/agent/decide
- **功能**：Agent 核心决策，接收直播多模态数据，返回提词建议
- **请求体**：包含直播状态、弹幕数据、商品数据、后台数据
- **响应**：提词指令和直播结构信息

### POST /api/v1/products
- **功能**：商品信息管理
- **请求体**：商品详细信息
- **响应**：创建结果

### POST /api/v1/compliance/check
- **功能**：检查文本是否合规
- **请求体**：待检查文本
- **响应**：合规检查结果

### POST /api/v1/rag/search
- **功能**：语义搜索知识库
- **请求体**：搜索查询和参数
- **响应**：搜索结果

## 注意事项

1. **API 密钥安全**：`.env` 文件包含敏感信息，请勿提交到 GitHub
2. **环境要求**：确保 Docker 版本 >= 20.10，Docker Compose 版本 >= 2.18
3. **资源配置**：建议服务器至少 4GB 内存，以支持所有服务正常运行
4. **网络连接**：确保服务器能够访问 LLM API 和向量生成 API

## 故障排查

- **服务启动失败**：检查 `LLM_API_KEY` 是否正确设置，查看日志：`docker-compose logs backend`
- **前端无法连接后端**：检查 `VITE_API_BASE_URL` 配置，确保后端服务正常运行
- **Qdrant 搜索失败**：检查 Qdrant 服务状态：`docker-compose ps qdrant`
- **数据库连接失败**：检查 MySQL 服务状态和连接字符串配置
